# Harness P0-1 and P1-2 Implementation Plan

> **For agentic workers:** Execute inline in the current workspace. Do not create a worktree, commit, stage, or dispatch subagents. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add protected configuration persistence and the documented six permission modes without breaking existing saved configuration or sessions.

**Architecture:** A generic lock/backup/atomic-write module protects application-owned files, while a config crypto module owns authenticated secret envelopes. Permission policy stays deterministic in `PermissionEngine`; asynchronous AI classification and parent approval routing remain in `ToolRuntime`/`LocalHarness` where async services already exist.

**Tech Stack:** TypeScript 7, Node.js 20 filesystem/crypto APIs, Zod 4, Vitest 4.

## Global Constraints

- Modify the current workspace directly; do not create a worktree.
- Do not commit or stage changes.
- Preserve unrelated dirty-worktree changes.
- Use test-first red/green cycles for behavior changes.
